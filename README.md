**[ НАЗВАНИЕ ПРОЕКТА ]**

“HappyHome”

**[ ПРОБЛЕМНОЕ ПОЛЕ ]**

Отсутствие единой и доступной всем членам семьи базы лекарств с указанием их срока годности и их назначения. Несвоевременное пополнение домашней аптечки, в связи с этим велик риски употребить просроченное лекарство в то время, когда оно очень нужно.  Ежедневной рутиной каждой семьи является приобретения продуктов питания и товаров первой необходимости. Часто случаются ситуации, когда нужные продукты никем не были куплены, но приобретено что-то другое в большем, чем нужно количестве, тем самым растрачивая семейный бюджет впустую. Из-за забывчивости и занятости члены семьи не соблюдают режим полива и удобрения или путают периодичность и способы полива для разных растений, в результате чего растения чахнут.

**[ ЗАКАЗЧИК / ПОТЕНЦИАЛЬНАЯ АУДИТОРИЯ ]**

Целевая аудитория: 

* Взрослые люди 30-40 лет, проживающие совместно с другими членами семьи и ведущие домашнее хозяство, объединенные общими бытовыми вопросами.

**[ АППАРАТНЫЕ / ПРОГРАММНЫЕ ТРЕБОВАНИЯ ]**

Kotlin - для нативного Android-приложения (фронтенд).
Python - для бэкенд-сервера (реализация логики, хранение данных, отправка push-уведомлений).
База данных - MySQL.

Первая версия приложения разрабатывается для Android.

* Android – (Android 7) и выше, 12 Гб свободного места, 3 Гб оперативной памяти

**[ ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ ]**

Приложение для личное/семейного использования. 
Не предназначено для крупных бизнесов или больших сообществ. 
Цифровой продукт, распространяемый через магазины приложений. 
Авторизация пользователей и функционал создания совместной (семейной) группы, добавление участников с правами. 
* Модуль 1. Умная аптечка. Добавление лекарства, заполнение карточки с указанием срока годности, уведомления об окончании срока годности. 
* Модуль 2. Совместные покупки. Регулярно обновляемый список продуктов и товаров первой необходимости. Добавление нового товара, назначение даты закупки товара, настройка периодичности добавления в список покупок, сортировка товаров.
* Модуль 3. Уход за растениями. Картотека растений: Для каждого цветка можно указать название, фото, особенности. График полива: Настраивается индивидуально для каждого растения. Оповещения. Ведение история ухода

**[ ПОХОЖИЕ / АНАЛОГИЧНЫЕ ПРОДУКТЫ ]**

1) Группа приложений для отслеживания геолокации и местоположения телефона ребенка. Не могут решить проблему ведения базы покупок и мониторинга выполнения нужных задач в кругу семьи.
2)  Совместный календарь с некоторыми дополнительными функциями (фотоальбом, книга семейных рецептов и т.д.).
3)  Приложение-помощник в рутинных делах, но зачастую проект решает одну задачу (например, планирование семейного бюджета), нужно ставить несколько приложений, что занимает место на телефоне, пользователь вынужден переключваться между разными приложениями.  

Анализ 3 программных продуктов, которые максимально приближены к заданному функционалу, показал, что:

* Cozi Family Organizer - содержит календарь-синхронизатор для всей семьи, но нет умного автодобавления рутинных действий, и также отсутствует умная аптечка. 
* PikaPika - позволяет назначить список задач и ответственных за их выполнение, приложение решает задачу делегирования обязанностей.
* Trello - набор функций на основе карточек, досок задач, команд. Не позволяет отслеживать состояние домашней аптечки, создавать по расписанию задания на приобретение товаров первой необходимости, бесплатная весрия очень ограничена по функционалу.
